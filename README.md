# 🎬 Game Ad Production Director

**把游戏买量创意从“一个想法”，推进到真正能制作、能审片、能投放、能迭代的成片方案。**

不是只负责想钩子的灵感工具，也不绑定 AI 剧情、AE 实机或某一种表现形式。它会先判断产品应该卖什么体验，再为纯实机、UGC、短剧、AI 影像、2D、3D、ASMR 或混合媒介选择可靠的生产路线。

## 为什么需要这套 Skill？

游戏广告最常见的问题不是“没有创意”，而是创意无法稳定走到成片：

- “前三秒很炸，但和后面的玩法没有关系。”
- “分镜写得很漂亮，AI、AE 和剪辑却不知道各自该做什么。”
- “AI 镜头能生成，角色、视线、道具和转场却接不上。”
- “实机看起来像自动播放，没有玩家输入、上下层关系或失败压力。”
- “画面做完才发现文字、安全区、画幅、时长或声音不符合投放要求。”
- “只做出一条成片，没有低成本变量，投放结果不好就只能推倒重来。”

这些并不是单个软件的问题，而是缺少一条从策略到交付的生产链路。

**Game Ad Production Director 把这件事变成一个可执行流程：**

```text
锁定需求 → 检查玩法与竞品证据 → 建立钩子与因果主干 → 选择制作形式
→ 输出逐秒分镜与资产契约 → 分路线制作 → 合成声音 → 投放质检 → 设计测试变量
```

它始终围绕一条主链路工作：

```text
阻止划走 → 让冲突一眼可懂 → 证明玩家操作 → 释放情绪回报 → 促使行动
```

## ✨ 它能做什么

| 阶段 | 交付能力 | 典型输出 |
|---|---|---|
| 创意策略 | 拆解受众矛盾、USP、竞品表达、前三秒钩子和留存悬念 | 创意方向、评分表、风险与备选方案 |
| 形式路由 | 根据产品、预算、周期和制作把握选择表现形式 | 纯实机 / UGC / AI 剧情 / 真人 / 2D / 3D / ASMR / 混合媒介 |
| 可执行分镜 | 把概念写成逐秒可制作镜头 | 时间码、动作、运镜、转场、声音、制作分工、交接帧 |
| AI 生产包 | 约束参考图、角色、场景、首尾帧和连续性 | 图片提示词、首尾帧视频提示词、全能参考提示词、失败重试规则 |
| 玩法与 AE | 还原玩家输入、合法操作、层级、遮挡、库存、失败与成功状态 | AE 实机方案、UI、点击反馈、消除光效、嵌套合成规范 |
| 成片组装 | 统一画面、节奏、字幕、声音、CTA 与平台安全区 | 剪辑节拍、BGM/SFX/VO 设计、合成与渲染规范 |
| 审片质检 | 从创意逻辑、玩法真实性和技术规格三层检查成片 | 缺陷清单、修改优先级、媒体 QC 报告 |
| 投放迭代 | 在不重做整条素材的前提下设计有效变量 | 钩子、冲突、节奏、回报、文案与 CTA 测试矩阵 |

### 装好后可以直接这样说

- “给这款三消游戏做 5 个能在一周内落地的 TikTok 创意。”
- “把选中的创意写成 15 秒竖屏分镜，并明确 AI、AE 和剪辑分工。”
- “为每个 AI 镜头生成图片提示词、首尾帧视频提示词和连续性约束。”
- “按照真实 Tile 玩法重构实机，检查遮挡、托盘、解锁和三消逻辑。”
- “从发行设计总监视角审一遍这条成片，只诊断，不修改源文件。”
- “把现有成片拆成三个低成本 A/B 测试版本。”

不需要先指定软件。Skill 会先确认要卖给观众的体验，再决定用 AI、AE、真人、2D、3D 或组合方案。

## 🧭 支持的制作路线

| 路线 | 适合什么目标 | 关键控制点 |
|---|---|---|
| 纯实机 | 快速证明玩法、追求产品真实性 | 输入可见、局面可读、爽点递进 |
| 失败挑战 | 用“差一步”“马上点错”制造强迫观看 | 失败压力真实，解法能被观众理解 |
| UGC / 口播 | 建立真人感、评价感和社交证明 | 前置结论、自然表演、字幕节奏 |
| 真人或 AI 短剧 | 用角色目标和危机扩大受众面 | 动机、视线、手部、道具和因果转场 |
| 2D 动效 | 强调图形设计、规则解释和品牌统一 | 手机端尺寸、节奏、信息层级 |
| 3D 展示 | 强调材质、空间、冲击和高规格视觉 | 资产成本、镜头连续性、产品识别 |
| ASMR / Oddly Satisfying | 放大整理、合并、清除与触感回报 | 运动曲线、细节拟音、节奏留白 |
| 元玩法 / 混合媒介 | 用外部冲突自然导入核心玩法 | 钩子与实机必须存在清晰因果 |

AI 剧情加实机只是其中一条路线，不是默认答案。中高风险方案必须保留一个钩子类型不同的备选方向。

## 🎯 参考案例

这些案例来自同一批 Tile 创意开发，但分别验证不同的钩子机制和制作路线。它们不是固定模板；复用时应保留“冲突—玩家操作—反馈—回报”的因果链，再替换题材、资产和测试变量。

### 案例 1：机场安检——行李箱里的 Tile

**路线：AI 角色短剧 + AE 实机 + ASMR**

- **前三秒钩子：** 暖色卡通浣熊急切看向 X 光安检机；行李箱即将进入扫描区域，危险来自“箱内塞满违禁式杂物”。
- **因果主干：** X 光照出拥挤的旅行道具 → 镜头切为俯视 Tile 棋盘 → 玩家逐件点击、凑齐三枚并释放下层物品 → 行李正常通过 → 浣熊开心拖箱旅行。
- **制作分工：** AI 负责机场角色表演、扫描机环境与结尾旅行镜头；AE 负责全部真实点击、托盘、遮挡解锁、三消、扫描光和 UI；剪辑负责 X 光声、软质弹跳、卡扣与结尾音乐释放。
- **关键约束：** 实机必须是竖屏俯视；下层物品需被上层真实压住并压暗；不是自动扫描消除，而是逐个找到、点击、入槽后三消。
- **可测试变量：** 浣熊先看机器或先护住箱子；3 秒或 4 秒切入实机；紧张警报型声音或舒缓 ASMR 型声音。

### 案例 2：A 还是 B？只有一步能活

**路线：纯实机 + 互动选择 + 失败挑战**

- **前三秒钩子：** 托盘已占 6/7，棋盘同时标出 `A`、`B` 两个可点目标，字幕只问 `A OR B?`。
- **玩法证明：** 先故意点错 A，让第七格触发失败；立即倒回，改点 B，解除遮挡并完成连续三消。
- **爽点设计：** 错误低沉、正确清脆；托盘从 6/7 快速降位，随后用 Combo ×2、×3 建立递进，而不是一次清空。
- **制作优势：** 全部可在 AE 内完成，适合低成本批量换棋盘；同一套主体只替换残局与首屏文案即可测试。
- **可测试变量：** `A OR B?` 对比 `DON'T TAP THE OBVIOUS ONE`；先展示错误结果或先留 1.5 秒给观众判断。

### 案例 3：戴耳机看完这一局

**路线：纯实机 + Oddly Satisfying + ASMR**

- **前三秒钩子：** 棋子微距特写，指尖按下后产生软塑料压缩与回弹；首轮三消在 3 秒内完整发生。
- **节奏结构：** 清晰点击 → 弧线入槽 → 三枚靠拢 → 深层 Pop → 下层棋子松开；在托盘 6/7 时停声 0.25 秒制造危机，再恢复更密集的触感节奏。
- **声音分层：** 指尖接触、棋子回弹、入槽 Whoosh、落槽接触、三消 Pop 与极轻低频分别控制，不依赖持续爆炸特效。
- **关键约束：** 镜头缓推不甩动；点击光效只覆盖局部；运动曲线必须有重量和摩擦感，避免像自动播放。
- **可测试变量：** 无 BGM 纯 ASMR；极弱 Lo-fi Pad；近景材质版与完整棋盘版。

### 案例 4：解冻玻璃球里的夏天

**路线：AI 微缩世界 + AE 实机 + 世界修复反馈**

- **前三秒钩子：** 盛夏海滩被冻在玻璃球里，微缩游客发抖，冰霜还在继续扩散。
- **因果主干：** 玩家在玻璃球底座棋盘完成火焰三消 → 冰层裂开并恢复一部分暖色 → 多轮真实遮挡解锁 → 海滩完全复苏。
- **制作分工：** AI 生成同一玻璃球的冻结首帧与解冻尾帧；AE 制作底座棋盘、冰裂遮罩、颜色恢复、托盘和 Combo；声音从寒风、冰裂逐层过渡到海浪与鸟鸣。
- **关键约束：** 环境变化只作为成功反馈，不能伪装成游戏内已有功能；玩家每次操作仍须符合真实 Tile 规则。
- **可测试变量：** 冰冻海滩对比枯萎花园；先展示最终暖色结果或先展示冰霜危机。

### 案例 5：一组三消，引爆整条多米诺

**路线：混合媒介 + 物理危机 + 节奏展示**

- **前三秒钩子：** 顶视桌面上多米诺快速倒向一杯果汁，最后一块在棋盘边缘停住。
- **因果主干：** 棋盘每完成一组三消，多米诺就推进一段；托盘到 6/7 时全场静音；正确消除改变路线，让多米诺绕开杯子并按亮终点开关。
- **制作分工：** AI 或 3D 负责桌面、多米诺开头和安全结尾；AE 控制实机、棋盘占位、路线反馈、光源启动与所有可读 UI。
- **关键约束：** 外部物理世界只放大操作结果，不替代玩法；多米诺与实机节拍必须逐次对应，避免看起来互不相关。
- **可测试变量：** 果汁杯危机对比玻璃摆件危机；连续节奏版对比 6/7 静音悬念版。

选择案例时，先判断当前素材最需要补足什么：**玩法可信度不足**优先案例 2，**感官爽感不足**优先案例 3，**题材记忆点不足**优先案例 1 或 4，**玩法与外部剧情缺少连接**优先案例 5。

## ⚙️ 工作原理

这套 Skill 不是一份固定模板，而是一层游戏广告生产决策系统。

### 1. 先找产品真相

识别真实玩家输入、状态变化、失败条件、爽点、产品识别元素与素材限制。竞品只作为市场表达证据，不作为可照抄的镜头表。

### 2. 再建立广告主干

每个方向必须回答五个问题：

1. 0.5—3 秒内，什么能阻止用户划走？
2. 什么悬念让用户继续看？
3. 哪个动作能真正证明玩法？
4. 完成后有什么感官或情绪回报？
5. 用户为什么现在就要下载？

钩子如果不能自然进入产品证明，就不进入生产。

### 3. 按制作风险选择工具

- AI 负责允许一定模糊度的视觉发明。
- AE、剪辑和 2D/3D 软件负责精确玩法、UI、文字、品牌资产、遮罩和节奏。
- 真人或 UGC 负责真实反应、口语信任和社交场景。
- FFmpeg/ffprobe 负责媒体检查、音频分析、转码和最终验证。

### 4. 用交付契约连接每一环

每个镜头必须写清：观众任务、画面动作、制作方式、输入资产、输出规格、声音、验收标准和尾帧连续性。跨 AI、真人、AE 或 3D 时，明确准确的交接帧或交接动作。

### 5. 最后才进入投放质检

完成并不等于文件成功导出。成片还要通过创意因果、玩法与连续性、画面与声音、平台规格、本地化和测试变量检查。

## 🔍 媒体质检

安装 FFmpeg 后，可对竖屏成片进行基础技术检查：

```powershell
python scripts/media_qc.py "final-ad.mp4" `
  --min-duration 10 `
  --expect-aspect 9:16 `
  --report "qc-report.json"
```

脚本用于检查媒体规格；创意因果、玩法合法性、视觉连续性和主观声音体验仍需按 [投放质检规范](references/publishing-qc.md) 审核。

## ✅ 成片质量门槛

| 门槛 | 必须满足 |
|---|---|
| 钩子 | 前三秒不依赖背景知识也能理解 |
| 因果 | 开场、玩家操作与结局属于同一条逻辑链 |
| 产品 | 核心玩法动作清晰，真实性足以支撑卖点 |
| 连续性 | 角色、透视、道具、光向和运动方向跨媒介成立 |
| 制作质量 | 无手部崩坏、UI 漂移、错误重叠、穿插、水印或占位文字 |
| 声音 | 关键动作有反馈，层次清晰且混音不削波 |
| 平台 | 画幅、时长、安全区、语言、CTA、编码和音频符合要求 |
| 可测试性 | 不重做全部资产也能生成有意义的变量 |

没有完成实际渲染、源文件或验证时，Skill 不会宣称“成片已完成”。

---

## 🧱 能力边界

- 默认维护真实玩法。只有用户明确授权时，才设计并标注伪玩法路线。
- 不会因为工具可用就强行使用 AI、AE、真人或 3D。
- AI 生成内容不负责精确文字、Logo、UI 对齐和严格玩法状态。
- 审片任务默认先诊断；没有获得修改授权时，不改动源工程。
- Tile 机场案例只用于说明因果转场和混合制作方法，不自动复用浣熊、机场、行李箱、X 光机、具体镜头或文案。
- 最终投放表现仍需由真实数据验证；创意评分不能替代留存、点击和转化结果。
