# 玩法、AE与合成规范

## 动画前先还原玩法

把游戏写成状态和转换：

```text
棋盘状态 -> 可见玩家输入 -> 托盘/状态更新 -> 匹配或失败检查 -> 反馈 -> 下一个可操作状态
```

定义合法输入、遮挡规则、库存容量、匹配数量、失败条件和完成条件。美术精度无法修复错误的状态机。

## 分层Tile玩法逻辑

对于分层Tile玩法：

- 道具在空间中互相压住，单纯上下行不能表达层级；
- 被覆盖物需要锁定，通常压暗，不能点击；
- 每次找到并点击一个露出的道具；
- 道具进入托盘时不能穿过其他棋子或UI；
- 第三个同类进入托盘后才触发三消；
- 上层消除后，一个或多个下层物品才被揭示；
- 托盘容量制造可读风险；
- 清空棋盘获胜，托盘填满且无法匹配时失败。

不要把三个相同的可点击道具直接并排放好来强行制造三消。需要展示发现、风险和释放。

## 保护权威坐标

- 把设计完成的棋子坐标视为玩法事实。
- 不要为了适配新背景而拉伸单个棋子坐标。
- 通过容器、UI底板、相机或预合成变换适配画面。
- 除非玩法改变，否则保持比例和相对遮挡关系。
- 审查移动路径，避免棋子与无关物体或UI穿插。

## AE工程结构

推荐使用可复用预合成：

```text
MASTER_9x16
  STORY_OR_HOOK
  GAMEPLAY
    BG
    BOARD
      DEPTH_BACK
      DEPTH_MID
      DEPTH_FRONT
    INPUT_FX
    TRAY
    PROGRESS
    GAMEPLAY_FX
  ENDING
  COPY_CTA
  AUDIO_GUIDE
```

检查既有工程时必须遍历嵌套预合成。图层可能位于背景、棋子、托盘、进度或特效预合成中，而不在主合成里。

## 确定性动画规则

- 一次只表现一个能够归因于玩家的输入。
- 每个操作包含预备、移动、落位、状态更新和反馈。
- 点击指示在接触后不要继续遮挡移动物体。
- 为托盘槽、文字和棋盘边缘建立保护区。
- 下层用亮度、缩放和景深变化解锁，不要无原因瞬移。
- 节奏经历建立、第一次证明、紧张、连击和释放，不能全程等间隔。
- ASMR方向延长预备和落稳，减少镜头噪声，加强细致拟音。
- 高能方向可以压缩空白，但必须保留能理解因果的关键帧。

## 特效分工

AI生成的光效图更适合作为设计参考或纹理底板。需要精确时机和复用的点击、匹配、扫描、进度和消除效果在AE中制作。

区分点击确认、移动轨迹、托盘落位、匹配/连击、解锁光、进度脉冲和最终清空。所有事件共用一种发光会削弱反馈层级。

## 审核节点

至少渲染并检查：首帧、第一次输入、第一次托盘更新、第一次匹配、中段锁定/解锁状态、临界失败、完成以及CTA尾卡。不能只凭一张截图批准布局。

## 脚本和桌面操作原则

- ExtendScript负责重复性的合成搭建、命名、变换、时间和批量修正；
- 桌面控制负责脚本无法覆盖的视觉验证、渲染对话框和软件操作；
- 结构性修改前保存版本化AEP；
- 脚本执行成功不等于画面正确，必须检查渲染帧；
- 保留用户修改和无关图层。
